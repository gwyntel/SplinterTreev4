fix: implement dynamic context window settings with backwards compatibility

- Add slash commands for managing context window size per channel
- Add legacy command support (!setcontext, !getcontext, !resetcontext)
- Add persistent storage for context window settings
- Update help documentation to reflect both slash and legacy commands
- Fix default context window behavior
- Add admin checks for context window management

The following commands are now available:
Slash commands:
/set_context_window - Set context window size (1-50 messages)
/get_context_window - View current context window size
/reset_context_window - Reset to default size (10 messages)

Legacy commands:
!setcontext - Set context window size (1-50 messages)
!getcontext - View current context window size
!resetcontext - Reset to default size (10 messages)
